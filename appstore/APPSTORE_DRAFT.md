## Privacy Policy Highlights

**For App Store Connect Privacy Section:**

### Data Collection
```
We do not collect ANY data from users. All expense data, receipts, and mileage records remain on your device.
```

### Data Usage
```
Not applicable - no data is collected or transmitted.
```

### Data Protection
```
All financial data is encrypted at rest using Apple's Data Protection APIs (AES-256 encryption). Data is protected by your device passcode or biometric authentication.
```

### Third-Party Access
```
No third parties have access to your data. We cannot access your data even if we wanted to—it never leaves your device.
```

### FinanceKit Disclosure
```
FINANCEKIT TRANSACTIONS (Optional Feature)

When you use the "Import Transactions" feature:
• You authorize the app to access financial data from Apple Card, Apple Cash, and participating banks through Apple's FinanceKit API
• Transaction data (amounts, merchants, dates) is stored locally in the app's encrypted database on your device
• Transaction data is NOT transmitted to our servers or any third parties
• You can revoke FinanceKit access at any time in iOS Settings > Privacy & Security > Financial Data
• FinanceKit is only available on iOS 17.4+ in the United States

The app accesses FinanceKit data solely to import transactions you select into your local expense records.
```

### User Rights
```
You have complete control over your data:
• Export all data to Excel at any time
• Delete individual expenses or all data
• No account creation required
• No data retention policies (we don't have your data)
```

## App Review Notes (For Submission)

```
FINANCEKIT USAGE

This app uses Apple's FinanceKit API to allow users to import transactions from Apple Card, Apple Cash, and participating banks.

FinanceKit is used to:
1. Allow users to select specific transactions using Apple's transaction picker UI
2. Import selected transaction data (amount, merchant, date, category) into the app's local database
3. Automatically categorize imported transactions for expense tracking

FinanceKit Requirements Met:
✓ App is in Finance category
✓ App provides financial management tools (expense tracking, tax reporting, budgeting)
✓ FinanceKit is optional - app fully functional without it
✓ All transaction data stored locally (not transmitted to servers)
✓ Users have full control over which transactions to import

TESTING INSTRUCTIONS

1. Launch app - no onboarding, goes straight to expense list
2. Tap "+" FAB to add expense
3. Test receipt capture: Tap camera icon, grant camera permission, capture receipt
4. OCR should extract text from receipt (requires real device)
5. Test mileage tracking: Navigate to "Mileage" tab, tap "+" to add entry
6. Test export: Tap menu (3 dots) > Export > Excel
7. Test FinanceKit (iOS 17.4+ only): Tap "Import" button, grant permission, select transactions

TEST CREDENTIALS: Not applicable (no login required)

DEMO VIDEO: [Link to demo video if available]
```

## Notes for Review

### Strengths of This Draft
- Clear privacy positioning (differentiator)
- Emphasis on offline/local-first capability
- Tax-ready features highlighted (target audience: freelancers/business owners)
- FinanceKit clearly explained as optional iOS 17.4+ feature
- Security messaging (bank-level encryption)
- Accessibility commitment mentioned

### Potential Concerns
- "Bank-level encryption" claim needs legal review
- FinanceKit availability limited to US (iOS 17.4+) and UK (iOS 18.4+) - clearly disclosed
- Should verify all claimed features are implemented in v1.0
- May want to soften "we can't access your data" to "we don't collect or access your data"

### Recommendations for Next Steps
1. Legal review of security and encryption claims
2. Validate all features listed are in v1.0 build
3. Prepare screenshots showcasing key features
4. Create demo video showing receipt capture and FinanceKit import
